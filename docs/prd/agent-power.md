# Tipi 浏览器插件 — AI 智能助理功能需求文档 (PRD)

## 1. 产品定位与设计哲学

AI 智能助理是 Tipi 插件的**可选增强模块（Opt-in Feature）**。它将 Tipi 从一个“字面量历史搜索工具”升级为一个“具备推理和反思能力的赛博助理”。

在引入 AI 的同时，必须死守 Tipi 的核心设计哲学：

* **隐私至上（Privacy First）**：使用用户私有的 DeepSeek API Key，数据直接从浏览器后台发往 DeepSeek 官方接口，不经过任何第三方中转服务器，100% 保护浏览历史隐私。
* **极致轻快（Minimal & Fast）**：UI 注入必须轻量，不拖慢宿主网站的渲染。只有在用户激活 AI 时才初始化相关模块。
* **键盘优先（Keyboard-First）**：完美支持全键盘操作（打开、关闭、发送、焦点切换），确保不破坏高频用户的操作流。

## 2. UI / UX 交互规范

### 2.1 悬浮球 (Floating Bubble)

* **触发与状态**：默认常驻在网页右下角（避开常见的返回顶部按钮，如距离右边 24px，底部 80px）。
* **样式规范**：
* 采用 Tipi 经典的纸张色系（`--color-surface` 作为背景，`--color-primary` 作为主体色）。
* 鼠标悬停（Hover）时呈现微弱的阴影加深或缩放动画（Scale 1.05）。
* 点击悬浮球，展开/收起对话面板。


* **不干扰原则**：通过 Shadow DOM 注入，设置 `pointer-events: auto`，但确保其边界盒（Bounding Box）不遮挡宿主网页的点击。

### 2.2 对话面板 (Chat Window)

* **展开样式**：点击悬浮球后，在悬浮球上方或左侧展开一个优雅的卡片面板（建议宽度 380px - 420px，高度固定 500px 或自适应）。
* **快捷键唤醒**：支持全局快捷键 `Option + I` (macOS) / `Alt + I` (Windows) 直接在当前页面唤醒面板。
* **组件结构**：
1. **Header**：Tipi AI 标志及一个“清除对话”与“关闭”按钮。
2. **Message Area**：对话历史滚动区域。
* 用户消息：靠右，浅色背景。
* AI 消息：靠左，支持流式（Streaming）Markdown 渲染。
* **历史记录卡片（Tool Outputs）**：当 AI 成功调用 Tipi 搜索并命中历史记录时，以极其优雅的“纸张卡片”形式独立嵌入在对话流中（包含 Favicon、网页标题、高亮 URL、访问时间）。点击可直接在新标签页打开。


3. **Input Area**：底部固定的输入框。
* 支持多行文本输入。
* 输入框激活时，边框颜色变为 `--color-primary`。





### 2.3 键盘交互规范 (Keyboard-First)

* **唤醒即聚焦**：无论是点击悬浮球还是通过快捷键打开面板，输入框必须**自动聚焦（Auto-focus）**。
* **发送操作**：按下 `Enter` 键直接发送消息；按下 `Shift + Enter` 换行。
* **快捷关闭**：在面板处于激活状态时，按下 `Esc` 键立刻隐藏面板，并将焦点还给宿主网页。

## 3. 核心功能与业务逻辑

### 3.1 智能化历史 RAG 检索

* **用户意图识别**：用户可以用自然语言提问（例如：“我昨天下午看的那篇关于 Tailwind v4 的博客在哪？”）。
* **参数提取**：Agent 需通过 System Prompt 引导，将模糊的用户提问转化为适合本地字面量检索的关键词（如提取出 `Tailwind v4`）。

### 3.2 自省与多轮重试机制（Self-Correction Loop）

这是利用 LangGraph.js 实现的核心业务壁垒：

1. **首次检索**：Agent 提取关键词并调用 Tipi 本地搜索工具。
2. **空结果反思**：如果 Tipi 工具返回 `[]`（未匹配到任何历史），Agent 不能直接回答“没找到”，而是必须进入**自省节点（Reflection Node）**。
3. **重试策略**：
* 重试次数限制为 **2 次**。
* 在自省中，Agent 需扩大检索范围或寻找同义词（例如：将 `Tailwind v4 博客` 拆解为更宽泛的 `Tailwind` 或 `v4`），然后自动发起第二次 Tipi 工具调用。


4. **最终输出**：如果重试后找到结果，或者达到重试上限，Agent 组合最终的上下文，向用户流式输出解答。

### 3.3 API Key 管理与隐私保障

* **免配置引导**：如果用户尚未配置 DeepSeek API Key，点击悬浮球或唤醒面板时，面板应展示一个精致的“引导配置”界面，提示用户前往 Tipi 设置页配置，或者直接在弹窗内提供 Key 输入框。
* **本地加密存储**：API Key 必须通过 `chrome.storage.local` 存储。
* **严禁越权**：Background 在调用 DeepSeek 接口时，必须直接请求官方 Endpoint (`https://api.deepseek.com`)，不允许将数据流经任何中转代理（除非用户自定义了 Base URL）。

## 4. 边界情况与异常处理 (Edge Cases)

| 场景 (Scenario) | 预期行为 (Expected Behavior) | UI 表现 (UI Specification) |
| --- | --- | --- |
| **网络断开/超时** | 超过 10 秒无响应或 fetch 抛出 Network Error。 | 停止流式流，在聊天区域显示错误卡片：“网络连接失败，请检查网络设置”，并提供“重试”按钮。 |
| **API Key 额度耗尽 / 无效** | DeepSeek 接口返回 401 或 402 错误码。 | 气泡框提示：“API Key 无效或余额不足，请前往 Tipi 设置页检查”。 |
| **彻底无匹配历史** | 经过 2 次换词重试后，Tipi 本地索引依然返回空。 | AI 应该委婉且人性化地回答：“在您最近 90 天的历史记录中没有找到相关网页。您可以尝试提供更准确的名称，或者去 Tipi 设置页扩大历史同步的窗口。” |
| **在受限网页上** | 用户处于 `chrome://`、`edge://` 或新标签页等 Extension 无法注入 Content Script 的页面。 | 悬浮球不显示（Extension 原生限制）。用户可以通过点击浏览器右上角的 Tipi 插件图标，在 Popup 窗口中照常使用 AI 对话（优雅降级）。 |

## 5. 验收标准 (Acceptance Criteria)

1. **响应速度**：从用户点击 `Enter` 发送，到后台 DeepSeek 吐出第一个 Token（首字响应），在网络正常情况下耗时应小于 800ms。
2. **样式隔离**：在 GitHub、知乎、Twitter 等各种复杂 CSS 环境的网站下测试，Tipi 悬浮球和对话框的字体、边框、背景均不受宿主网站样式污染。
3. **循环有效性**：人为制造一个模糊问题（如故意输错一个字母），通过 Background 的 `console.log` 能够清晰观察到 LangGraph 确实跑了 `Agent -> Tool -> Check -> Retry Agent -> Tool -> Respond` 的有环流程。
